Laravel Integration

Overview

This documentation manual will explain how to implement Keenthemes designed admin dashboard theme "Metronic" in Laravel application.

Requirements

  • composer
  • php
  • node / npm
  • Metronic theme

Installing Laravel

We will use Laravel command line installer (click here to read how to get it).

laravel new laravel

After Laravel finished installing, move to freshly installed application directory:

cd laravel

And open project in Your favorite editor (we will use Sublime).

Preparing Metronic theme assets

Purchase and Download Metronic theme here and extract package using Your favorite archive manager (i.e 7zip).

Navigate to theme folder and select desired layout (we will use demo1)

theme/demo1/src

Copy assets folder and place it under Your freshly installed Laravel applications resources directory so it looks like this

resources/assets

Preparing dependencies in Laravel package.json

Open file package.json which is located in Your Laravel application root directory. It should look like this:

{
    "private": true,
    "scripts": {
        "dev": "npm run development",

        "watch-poll": "npm run watch -- --watch-poll",

Compiling Assets

Now we are ready to prepare and configure our assets management using Laravel Mix. Let's open file webpack.mix.js from application root. It looks like this by default:

const mix = require('laravel-mix');

/*
 |--------------------------------------------------------------------------
 | Mix Asset Management
 |--------------------------------------------------------------------------
 |
 | Mix provides a clean, fluent API for defining some Webpack build steps
 | for your Laravel application. By default, we are compiling the Sass
 | file for the application as well as bundling up all the JS files.
 |
 */

mix.js('resources/js/app.js', 'public/js')
    .sass('resources/sass/app.scss', 'public/css');

We will modify this file to use Metronic assets (the ones we copied to resources/assets directory) and compile them down to our public folder.

public/js/scripts.bundle.js

Well done. Now let's add our initial styles from Metronic theme webpack. Open file:

theme/demo1/tools/webpack/styles.js

Which looks like this:

"use strict";

/**
 * Define the output of this file. The output of CSS and JS file will be auto detected.
 *
 * @output css/style.bundle

require("../../src/assets/sass/style.scss");

Here we can see that theme is loading single SCSS file and compiles it to css/style.bundle.css. We will do same in Laravel by adding this file path in webpack.mix.js and also include important part of theme - skins.

Important! Before building our styles we will need to edit file

resources/assets/sass/style.scss

And in line 20 (or around) find this

@import "../../../tools/node_modules/bootstrap/scss/bootstrap";

and replace it with

@import "node_modules/bootstrap/scss/bootstrap";

This means we will be referring to our node_modules folder instead of Metronic theme folder. Save file and now we are ready to compile our default assets.

And NOW we are ready to compile our complete set of assets.

npm run dev

Finally we see our new css style and skins files in public folder.
